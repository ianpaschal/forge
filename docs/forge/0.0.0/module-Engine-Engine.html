    Methods

    loop()

    This function will eventually be renamed 'update'. This is because each system actually controls its own update loop.

    This allows systems to be more deterministic. Some systems which use fixed timesteps get called on a regular basis. Although they may not execute exactly on schedule, so time is accumulated and then simulated in fixed steps, with a step being removed from the accumulator each time.

    Variable step systems, however, simply collect accumulated time and simulate some action based on that variable delta and eventually remove all accumulated time each loop.

    Systems which use fixed steps:

    • Resources
    • Physics
    • Comabat
    • Ai

    Systems which use variable steps:

    • Animation
    • Sound

    See http://gameprogrammingpatterns.com/game-loop.html
